Образ памяти
    указатели на сегменты кода
        данных из стека
        указатели на таблицы страниц
    сигналы
        маски указывающие какие сигналы игнорируются
        какие перехватываются
        какие временно заблокированы
        какие находятся в процессе доставки
    разные
        текущее 
        событие ожидания процесса
        время до истечения интервала будильника
        PID-процессор
        PID-родительского процесса
        идентификатор пользователя в группе

В структуре пользователя содержится информация, которая мигрируется, когда в процессе нет физической памяти и он не выполняется, 
    поэтому информация сигналов должна зраниться в таблице процессов постоянно находящихся в памяти, даже когда процесс не присутствует в памяти

Данные хранящиеся в структуре пользователя включают:
    машинные регистры
    состояние системного вызова
    таблицы дескрипторов файлов
    учетная информация
    стек ядра

Последовательность создания процесса
    когда выполняется системный вызов fork 
    выхывающий процесс обращается в ядро
    ищет свободную ячейку в таблице процессов, в которую можно записать данные о дочернем процессе
    системный вызов копирует туда информацию из ячейки родительского процесса
    выделяет память для сегментов данных из стеков дочернего процесса
    копирует соответсвующий сегмент родительского процесса
    структура пользователя копируется вместо со стеком
    сегмент кода может либо копироваться, либо использоваться совместно, если он доступен только для чтения

Если поток ядром не поддерживается реализация осуществляется в пространстве пользователя
Для реализации потоков используется системный вызов clone
